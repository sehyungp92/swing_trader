# ETRS vFinal Backtest Pipeline -- Claude Code Execution Guide

Instructions for Claude Code to execute the full backtesting pipeline with
**reduced trial counts** and **5-year data window** for faster iteration.
Each phase produces output that must be captured and recorded before proceeding
to the next.

All results are appended to `backtest_results.md` as each phase completes.

**Key design choice:** The strategy trades 1 share per signal for all ETFs.
This makes P&L directly comparable to a buy-and-hold benchmark of 1 share
held for the entire 5-year period.

### Step 2.3: Compute buy-and-hold benchmarks

For each ETF, compute the return from holding 1 share for the full data period
and for the common-overlap period. This is the benchmark the strategy must beat:

```bash
python -c "
import pandas as pd
from pathlib import Path

print('=== Buy-and-Hold Benchmark (1 share) ===')
data_dir = Path('data/raw')
symbols = ['QQQ','GLD','USO','IBIT']
for sym in symbols:
    f = data_dir / f'{sym}_1d.parquet'
    if not f.exists():
        print(f'{sym}: MISSING')
        continue
    df = pd.read_parquet(f)
    # Full period
    first = df.iloc[0]['close'] if 'close' in df.columns else df.iloc[0]['Close']
    last = df.iloc[-1]['close'] if 'close' in df.columns else df.iloc[-1]['Close']
    pnl_full = last - first
    ret_full = pnl_full / first * 100
    years = (df.index[-1] - df.index[0]).days / 365.25
    cagr_full = ((last / first) ** (1 / years) - 1) * 100 if years > 0 else 0
    print(f'{sym} full ({df.index[0].date()} - {df.index[-1].date()}):')
    print(f'  Buy={first:.2f}  Sell={last:.2f}  P&L=\${pnl_full:.2f}  Return={ret_full:.1f}%  CAGR={cagr_full:.1f}%')
    # Common overlap (2024-02-01+)
    df2 = df[df.index >= '2024-02-01']
    if len(df2) >= 2:
        f2 = df2.iloc[0]['close'] if 'close' in df2.columns else df2.iloc[0]['Close']
        l2 = df2.iloc[-1]['close'] if 'close' in df2.columns else df2.iloc[-1]['Close']
        pnl2 = l2 - f2
        ret2 = pnl2 / f2 * 100
        y2 = (df2.index[-1] - df2.index[0]).days / 365.25
        cagr2 = ((l2 / f2) ** (1 / y2) - 1) * 100 if y2 > 0 else 0
        print(f'  Overlap ({df2.index[0].date()} - {df2.index[-1].date()}):')
        print(f'  Buy={f2:.2f}  Sell={l2:.2f}  P&L=\${pnl2:.2f}  Return={ret2:.1f}%  CAGR={cagr2:.1f}%')
    print()
"
```

baseline across all symbols):

- **RELAX**: Mixed results (one metric improves, another degrades)

## Phase 5: Walk-Forward Validation (Reduced Trials)

**Goal:** Validate parameter stability with expanding-window walk-forward.
**Estimate:** ~16-27 hours (each fold runs its own internal optimization at
500 coarse + 150 refine by default within WalkForwardValidator).

Note: the walk-forward CLI does not expose `--n-coarse`/`--n-refine` for the
per-fold internal optimization. The default per-fold optimization uses 500
coarse + 150 refine (defined in `WalkForwardValidator.__init__`). This is the
minimum recommended for meaningful results.

### Step 5.1: ETF universe walk-forward (12-month windows)

With ~5 years of data for QQQ/GLD/USO, expect ~3 folds. IBIT will have fewer
folds due to its shorter history.

```bash
python -m backtest.cli walk-forward --test-months 12
```

Long-running: use background execution with periodic checks.

The output format:

```
=== Walk-Forward Results ===
Folds: N
Avg test score: X.XXXX
Avg test Sharpe: X.XX
Positive folds: XX%
Degradation ratio: X.XX
Robustness: PASS/FAIL
  - {failure reason, if any}
  - ...

Per-fold results:
  Fold 0: train=YYYY-MM-DD-YYYY-MM-DD test=YYYY-MM-DD-YYYY-MM-DD train_score=X.XXXX test_score=X.XXXX test_sharpe=X.XX trades=N
  ...
```

The framework now automatically enforces robustness constraints:
- Positive folds >= 60%
- Max DD per fold <= 25%
- OOS trades/month >= 1.0
- Degradation ratio >= 0.30

### Step 5.2: Health check assessment

Use the automated `Robustness: PASS/FAIL` output as the primary signal, then
evaluate against these additional human-readable criteria:

| Metric | Healthy | Concerning | Failing |
|--------|---------|------------|---------|
| Degradation ratio | > 0.5 | 0.3 - 0.5 | < 0.3 |
| Positive folds | > 60% | 40-60% | < 40% |
| Avg test Sharpe | > 0.5 | 0-0.5 | < 0 |

Assign a status: PASS (all healthy + robustness PASS), MARGINAL (any concerning),
FAIL (any failing or robustness FAIL).

## Reduced-Trial Budget

| Phase | Full Pipeline | This Pipeline | Savings |
|-------|--------------|---------------|---------|
| Data download | 8 symbols, 5-10Y each | 4 ETFs, 5Y each | ~60% |

| Ablation | 15 single + 3 variant = 18 tests | 13 single + 3 variant = 16 tests | ~11% |
| Optimization (39 params) | 1000 + 300 = 1300 trials | 100 + 50 = 150 trials (parallel) | ~87% |
| Walk-forward per-fold | 500 + 150 = 650 trials | 500 + 150 = 650 trials | 0% (minimum) |
| Walk-forward folds | 3-6 micro + 5-6 full | ~3 ETF only | ~50-70% |
| Full-size cross-validation | 1300 trials + runs | per-instrument ETF only | ~90% |
| 6-month windows | 6+ folds | skipped | 100% |
| **Total CPU time** | **~7-10 days** | **~1-2 days** | **~80%** |
